1. Pre-Steps: Preparing Your Photos (IMPORTANT)

Before creating a report, it is critical to prepare your site photos. Following these steps ensures report consistency, manageable PDF file sizes, and proper archiving on the company server.

Step 1: Crop Photos to a 4:3 Aspect Ratio

All photos in the final PDF report are formatted to a 4:3 aspect ratio (a standard landscape format). While the application will automatically crop your images to fit, doing this manually beforehand gives you full control over the final composition.

  • Use any standard photo editor (like Windows Photos, macOS Preview, or a free online tool) to crop your images.
  • Set the aspect ratio to 4:3 and frame your subject as desired.

Step 2: Resize Photos to Reduce File Size

High-resolution photos can create extremely large PDF files that are difficult to email and store. You must create resized versions of your photos for use in the reporting application.

  • A good target size is a maximum width of 1024 pixels. This provides excellent quality in the PDF without bloating the file size.
  • Use a photo editor's "Resize" or "Export" function to create these smaller copies. Be sure to save them as a separate set of files so you still have the full-size originals.

Step 3: Upload Photos to the X-Terra Server

Proper file management is essential. After preparing your photos, you must upload both the full-size originals and the resized versions to the appropriate project folder on the X-Terra server. This ensures we have high-quality originals archived and that your resized photos are backed up.

2. Getting Started with the Application

Once your photos are prepared and uploaded, you can begin creating your report.

Creating a New Report

From the main landing page, you can start a new, blank report by clicking one of the three main options shown below.

  • Click Photographic Log for a standalone report containing only photos and descriptions.
  • Click Daily Field Report for a comprehensive report that includes detailed activity logs and an integrated photo log.
  • Click Sask Power Daily Field Report for the specialized DFR format required for SaskPower projects.

Opening an Existing Report

You have two ways to open a project:

  • Recent Projects: The landing page lists your most recent work. Simply click on a project to open it. You can use the search bar to filter this list by project name or number.
  • Open From File: In any report view, click the Open button to browse your computer for a project file (.plog, .dfr, or .spdfr). You can also open files by double-clicking them if the application was installed via the installer.

3. Creating a Daily Field Report

The DFR includes all the features of a Photo Log, plus sections for detailed daily notes.

A Note on the Text Editor

The text fields for activities and observations use a special editor designed for clarity. You may notice a difference between how text looks in the app versus the final PDF.

  • In the App: The text is formatted with a "hanging indent." This means the bullet point is slightly to the left of the text. This is intentional and helps make long lists of activities easier to read and edit on screen.
  • In the PDF: For a professional and clean final document, the exported PDF aligns the bullet point and the text flush left. This is the standard formatting for formal reports.

In summary: The indented view in the app is for your convenience while editing. The aligned view in the PDF is the correct, final format. What you see in the app is not a formatting error.

Using the Bullet-Point Editor

  • The editor automatically starts with a bullet point (-). If you delete it, simply type a hyphen to begin a bulleted list.
  • Press Enter to create a new bullet point on the next line.
  • You can add detailed, multi-line descriptions for each activity.

4. The Photographic Log

Both the standalone Photo Log and the integrated DFR photo log function identically.

  • Project Header: Fill in all the details at the top. This information will appear on every page of the final PDF.
  • Adding Photos: Click the Add Photo (or Add Site Photo) button to add a new entry.
  • Photo Details: For each entry, fill in the Date, Location, and Description. The Photo Number is automatically assigned and updated if you reorder photos.
  • Uploading Images: Click the camera icon area to select one of your resized image files (JPG or PNG) from your computer.
  • Managing Photos: Use the up/down arrows to reorder photos and the trash can icon to delete an entry.

5. Saving and Exporting

The header contains the primary action buttons for managing your files.

Save Project vs. Save to PDF: What's the Difference?

This is the most important concept for managing your work.

  • Save Project: This saves your work-in-progress to a special editable source file (.plog, .dfr, .spdfr). This file contains all your data and can be re-opened in the app to continue editing. This is how you save your work for later.
  • Save to PDF: This creates a final, non-editable PDF document. This is for sharing, printing, or submitting your finished report. You cannot open a PDF to edit it in this application. Always save your project file before creating the PDF.

6. Managing Your Data

How Data is Saved

When you save a project, it is automatically added to your Recent Projects list. All project data, including images, is stored locally in the application's dedicated storage on your computer. It is not sent over the internet, ensuring your data remains private.

Recent Projects: Removing vs. Deleting

In the Recent Projects list, you can click the vertical ellipsis (three dots) to manage an entry:

  • Remove from recent: This only removes the project from the list on the landing page. The project data itself remains stored on your computer.
  • Delete project permanently: This completely deletes the project and all its associated images from your computer's local storage. This action cannot be undone. Only use this if you are certain you no longer need the editable project file.
